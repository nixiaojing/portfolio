Data Overview


The datasets obtained from Zillow Research in Data Gathering Session are combined as one dataset with label of U.S. Region (West, Northeast, Midwest, Southeast, Southwest). The data cleaning and processing procedures for Naive Bayes analysis can be found in Data Cleaning session using R code. The processed dataset is shown in Figure 1. There are 70 records (different metropolitan regions)in the dataset with four numerical variables of ZORI, Sales price, List price, and Inventory.


Figure 1. Metropolitan region combined dataset overview. (Download).


Support Vector Machine


  1. General setting


    2. Support Vector Machine (SVM) is a supervised machine-learning modeling method based on regression analysis. It can only be used on qualitative variables. One SVM can construct a hyperplane to devide a dataset into two sectors. To classify a dataset into several sections, multiple SVM are usually applied. SVM is also a robust method that only some of the points determine the location of hyperplane, which also means it is not sensitive to the outliers. In addition to linear classification, combining with different kernel transformations, SVM can be applied to more complex (Wiki).


    For Zillow research combined data, there are in total four numerical variables of ZORI, Sales price, List price, and Inventory. The label is the U.S. Region of each record including West, Northeast, Midwest, Southeast and Southwest. As a supervised learning method, Support Vector Machine classifier need "label" to train the model first. After model is trained, a testing set is needed to evaluate the model performance. The data is devided into training set (n=53) and testing set (n=17). All R code used in Support Vector Machine analysis can be downloaded here.


    Three models with different kernel transformations (linear, polynomial, and Gaussian radial based function (RBF)) are conducted to compare the performance using same training and testing data. The best cost is adjusted to 5 using tune() function in R for all three kernels.



  2. Model Comparison


    1. Confusion Matrix

      2. Figure 2 shows the confusion matrix of the model predictions for all three models. The overall accuracy of the three models are 47% (8 out of 17), 29%(5 out of 17), and 47% (8 out of 17) for SVM with linear kernel, polynomial kernal, and RBF kernel. respectively. The one with polynomial kernal has worst performance compared to the other two. Results of SVM with RBF kernal are similar to Linear kernel. Midwest and West have best performance with 100% accuracy, while Northeast and Southwest has the worst with 0 accuracy.
      The results indicate that Polynomial kernel is not an approporiate kernel transformation for this dataset.


      Figure 2. Confusion Matrix comparison. a, Linear kernel; b, Polynomial kernel; c, RBF kernel.



Summary


Support Vector Machine works better on this dataset compared to Naive Bayes classifier, which is an indication of that SVM is a more robust method compared to Naive Bayes classifier. Besides, SVM can handle the correlation among features that violates the assumptions of Naive Bayes classifier. Moreover, different kernel transfromation performed various on this datset. Linear and RBF kernel are more compatible with this dataset compared to polynomial kernel. Polynomial kernel classifys all class into Southeast region, which does not make sense for this dataset.


Conclusion


From the result, Midwest and West region real estate markets, represted by all home inventory, median sales prices, median list prices, are different from other regions. Northeast market is close to Southeast real estate market. Southeast and Southwest real estate markets are more close to Midwest other than Northeast and West market. Similar to NB resutls, when investors choose location to invest, instead of considering metropolitan region, whether to invest on a market like Northest and West, or a markt like Southeast, Southwest, or Midwest can be another option to consider. Different region would have other feature determine the market, such as real estate regulation or other housing benifits. To investigate more, more data on different U.S. region are required.

Data Overview


Based on the news inventory obtained from News API (Data Gathering Session), full news content are obtained through online news source including Reuters, CNN, BBC and so on. The data cleaning and processing procedures for Support Vector Machine can be found in Data Cleaning session using Python code. Figure 1 shows the data before and after cleaning. In this session, to perform Support Vector Machine, Full news content are converted to document term matrix (DTM) using TfidfVectorizer showning in Figure 2. There are 51 news in the dataset.


To perform Support Vector Machine model, the labels are set as the keywords used to obtain the news (More details can be found at Data Gathering Session. There are three keywords including "home buying", "mortgage", and "real estate".


The word clouds for three labels can be found in Decision Tree Session . The words in the news with different labels are distinguished in the word cloud plots. Words like "treasury" and "rate" are dominated in mortage news. On the contrary, news with keyword of "real estate" seems focus more on oversee market such as China. "housing", "price", and "sales" are the dominated words in the home buying news.

Support Vector Machine


  1. General setting


    2. Support Vector Machine (SVM) is a supervised machine-learning modeling method based on regression analysis. It can only be used on qualitative variables. One SVM can construct a hyperplane to devide a dataset into two sectors. To classify a dataset into several sections, multiple SVM are usually applied. SVM is also a robust method that only some of the points determine the location of hyperplane, which also means it is not sensitive to the outliers. In addition to linear classification, combining with different kernel transformations, SVM can be applied to more complex (Wiki).


    For news content dataset, there are in total 15779 words (variables) used in the model. The label is the keywords used to obtain the news. 70% of the data are randomly selected as training dataset, while another 30% is the testing dataset. Three different SVM with various kernel transformations are conducted to compare the performance using same training and testing data. The code performing SVM can be download here.




  2. Model Comparison


    1. Confusion Matrix

      2. Figure 3 shows the confusion matrix of the model predictions for all three models. The overall accuracy of the three models are 69% (11 out of 16), 38%(6 out of 16), and 50% (8 out of 15) for SVM with linear kernel, polynomial kernal, and RBF kernel. respectively. SVM with linear kernel is the one perform the best. The one with polynomial kernal has worst performance compared to the other two. Both polynomial kernel and RBF kernel failed to predict data with "real estate" label. This is as same as the Naive Bayes result. The results indicate that Polynomial kernel is not an approporiate kernel transformation for this dataset.


      Figure 3. Confusion Matrix comparison. a, Linear kernel; b, Polynomial kernel; c, RBF kernel.



Conclusion


Based on the results, the text data can be modeled by support vector machine. The overall performance of SVM is better than Naive Bayes classifier considering both record and text data. For the news data, the real estate news is kind of mixed together with the other two. In other words, Mortgage news and Home buying news are distincted from each other, while they both are related to real estate. Many real estate news likely mentions words in mortgage news. This is also an indication of that how important "mortgage" is to real estate market. In the other session, regions are found to be important for home buyers and investors to make purchase. Thus, for home buyer and other investors, considering overall mortgage rate and local mortage rate is critical for the decision to purchase a home.
