Data Overview


The datasets obtained from Zillow Research in Data Gathering Session are combined as one dataset with label of U.S. Region (West, Northeast, Midwest, Southeast, Southwest). The data cleaning and processing procedures for Naive Bayes analysis can be found in Data Cleaning session using R code. The processed dataset is shown in Figure 1. There are 75 records (different metropolitan regions)in the dataset with 5 variables including four numerical variables of ZORI, Sales price, List price, and Inventory, and one categorical variables as Political standing.


Figure 1. Metropolitan region combined dataset overview. (Download).


Naive Bayes


  1. General setting


    2. Naive Bayes classifier is a supervised machine-learning modeling method based on Bayes's theorem. It can be used to handle and predict both qualitative and quantitative variables. It assumes the independence among the features. For discreted data, it calculates the probability of certain combination of features happens. For continues data, it can coupled with kernel density estimation to achieve higher accuracy level. (Wiki).


    For Zillow research combined data, there are in total five variables used in the model including four numerical variables of ZORI, Sales price, List price, and Inventory, and one categorical variables as Political standing. The label is the U.S. Region of each record including West, Northeast, Midwest, Southeast and Southwest. As a supervised learning method, Naive Bayes classifier need "label" to train the model first. After model is trained, a testing set is needed to evaluate the model performance. The data is devided into training set (n=57) and testing set (n=18). All R code used in Naive Bayes classifier analysis can be downloaded here.




  2. Model Results


    1. Confusion Matrix

      2. Figure 2 shows the confusion matrix of the model prediction. The model has poor performance with accuracy of 33% (6 out of 18). Midwest has best accuracy of 100%, while Southwest, West and Northeast has the lowest accuracy of 0, which means none of the prediction is correct. There are many cells with zero value, which could be an indication of that those pairs are not alike.


      Figure 2. Confusion matrix using the Naive Bayes classifier



    2. Model adjustment

      3. The feature importance code didn't come through with error message "Not compatible with requested type: [type=character; target=double]". Feature importance should work on both categorical variables and numerical variables. The reason might be that the data itself is not suitable for applying Naive Bayes classifier. As mentioned above, Naive Bayes classifier assumes there no correlation among features, which is not the case for this dataset.
      Figure 3 shows the correlation and distributions of the features. Some features are correlated such as ZORI, Sales, and List prices.


      Figure 3. Correlation among features.



      After reducing the features to only "ZORI", "Inventory", and "politics", the confusion matrix is shown in Figure 4. The overall accuracy is still 33%. However, the accuracy of West region is no 0 anymore.


      Figure 4. Adjusted model performance.



Summary


Naive Bayes classifier did't perform well on this dataset. There might be two reasons. First, the dataset is relatively small. This would affect the robust perfomance of the classification. One of the online discussions mentioned that Naive Bayes classifier can be affected by outliers, which might be similar for this dataset.


Conclusion


Naive Bayes classifier method has strict assumption on indenpendence. Not every dataset is suitable for applying this method. For Zillow Research dataset, although there are clear differences among the Sales/List prices in different U.S. regions (EDA), Naive Bayes classifier cannot separate the data into corrected classes. The reason might because of that the dataset is out of the application scope of Naive Bayes classifier.
From the result itself, all features considered, Midwest region real estate market, represted by all home inventory, median sales prices, median list prices, and political standing, is different from other regions. Northeast, and West region real estate markets are most similar. Southeast and Southwest real estate markets are more close to Midwest other than Northeast and West market. Thus, when investors choose location to invest, instead of considering metropolitan region, whether to invest on a market like Northest and West, or a markt like Southeast, Southwest, or Midwest can be another option to consider. Different region would have other feature determine the market, such as real estate regulation or other housing benifits. To investigate more, more data on different U.S. region are required.

Based on the news inventory obtained from News API (Data Gathering Session), full news content are obtained through online news source including Reuters, CNN, BBC and so on. The data cleaning and processing procedures for Naive Bayes can be found in Data Cleaning session using Python code. Figure 1 shows the data before and after cleaning. To perform Naive Bayes classifier, the labels are set as the keywords used to obtain the news (More details can be found at Data Gathering Session. There are three keywords including "home buying", "mortgage", and "real estate".

Conclusion


Based on the results, the text data is more easy to be modeled by decision tree model compared to record data. One of the reasons may be that the words associated with other clearly indicate the topic of the article. For example, article contains "treasury" is highly related to key words "mortgage". In other words, when people use "mortgage" to find articles, most of them would mention "treasury". Similarly, when people use "home buying" to find articles, most of them would mention "sales" and "prices". There is a phrase "grew trillion" is associated with home buying news and real estate news. This is predictable because such magnitude can only happen on the housing market instead of description of change of mortgage rate.
This conclusion is similar with decision tree analysis. Indeed, when decision tree analysis failed to predict record data in Decision Tree session, one of the reason might be that it fails to use the interaction among features. Here, Naive Bayes classifier also works better on text data, which could be an indicator that the correlation among features play important role in modeling. Thus, no matter what methods are applied to the data, checking the assumptions of the methods and find a approporiate one is the most important thing for data scientist.
